# Field Flow 備案計畫（Backup Plan）

本文件定義 AI、Supabase、網路等關鍵基礎設施不可用時的人工備案流程。

---

## 1. AI 助手不可用

### 症狀

- 法法使用 AI 助手時看到「AI 目前正在午休中」
- API 回傳 503 或 timeout

### 可能原因

- OpenAI API 服務中斷
- API key 過期或額度用盡
- 網路不穩導致 API timeout（15 秒）

### 備案流程

1. **法法端**：AI 助手不可用時，畫面固定顯示「AI 目前正在午休中」
2. **法法仍可使用正式提問功能**：不依賴 AI，直接在 Q&A 頁面提問
3. **努努加快 Q&A 回覆節奏**：
   - 指定 1-2 名努努專責 Q&A 回覆
   - 努努後台 Q&A tab 篩選「待回答」，逐一回覆
4. **現場口頭補充**：若問題量大，努努可透過麥克風統一回覆常見問題
5. **知識庫仍可閱讀**：法法可在 Q&A 中搜尋已回答的問題

### 預防措施

- Demo 前確認 API key 有效且額度足夠
- 確認 AI_MODEL 設定正確（預設 gpt-4o-mini）
- 提前準備充足的 Q&A seed data，降低對 AI 的依賴

## 2. Supabase 不可用

### 症狀

- 頁面載入失敗或顯示「系統暫時無法連線，請稍後再試；若持續發生請改走人工備案」
- 登入功能失效
- Q&A、報到、午餐操作均無法執行

### 可能原因

- Supabase 服務中斷（罕見，可查 status.supabase.com）
- Supabase 免費方案 rate limit（超過連線數或 API 請求數）
- SUPABASE_URL 或 SUPABASE_SERVICE_ROLE_KEY 設定錯誤

### 備案流程

1. **報到**：
   - 改用紙本名單 + 手動打勾
   - 準備列印版學員名單（至少含：編號、姓名、飲食別）
   - 名單由報到組負責

2. **午餐**：
   - 改用紙本勾選，依名單上的飲食別發放
   - 報到時順便標記飲食別

3. **Q&A**：
   - 法法口頭提問或舉手
   - 努努現場回覆
   - 可用白板或投影片收集常見問題統一回答

4. **Agenda**：
   - 改由主持人口頭宣布目前進行到哪個階段
   - 投影片或白板顯示時間表
   - 可列印紙本 agenda 發給學員

5. **Dashboard**：
   - 改由報到組手動統計報到與午餐人數
   - 定期口頭回報給統籌

### 預防措施

- Production Supabase 使用獨立 instance（不與 dev/preview 共用）
- 確認 Supabase project 方案足夠活動規模（100 學員 + 12 努努）
- 活動前一天驗證所有 Supabase 連線正常
- 準備紙本名單作為最後手段

## 3. 網路完全中斷

### 症狀

- 所有網頁功能不可用
- 手機無法連線至 Field Flow

### 可能原因

- 活動場地 Wi-Fi 故障
- 行動網路收訊不良（戶外場地）

### 備案流程

1. **完全離線備案**：啟動 Part 2（Supabase 不可用）的全部紙本流程

2. **Agenda 同步**：
   - 主持人定時口頭宣布
   - 張貼紙本或投影片 agenda

3. **Q&A**：
   - 口頭提問 + 現場回答
   - 可用便利貼收集問題，集中回覆

4. **報到 / 午餐**：
   - 完全改用紙本

5. **網路恢復後**：
   - 努努將紙本報到資料補登到系統
   - 補登午餐領取紀錄
   - Q&A 可事後補建

### 預防措施

- 確認場地 Wi-Fi 容量與穩定性
- 準備至少一台 4G/5G 熱點裝置作為備用
- 所有紙本備案物品事先列印準備好

---

## 4. Vercel 部署不可用

### 症狀

- Production URL 無法載入（5xx 錯誤或 DNS 失敗）

### 可能原因

- Vercel 全域服務中斷（極罕見，可查 vercel.com/status）
- 部署配置錯誤

### 備案流程

1. 查看 Vercel 狀態頁確認是否為全域問題
2. 若為部署錯誤：
   - 在 Vercel Dashboard 回退到前一個成功的部署版本
   - 或從 GitHub main branch 重新觸發部署
3. 若 Vercel 全域中斷：
   - 啟動完整離線備案（同 Part 3）

### 預防措施

- Demo branch freeze 後禁止未經審查的 merge 到 main
- 確認 production deployment 成功且可正常使用
- 記錄最近一次成功的 deployment URL

---

## 5. 備案物品清單

活動前需準備以下紙本 / 實體備品：

| 項目 | 數量 | 負責人 | 備註 |
|------|------|--------|------|
| 紙本學員名單（含編號、姓名、信箱、飲食別） | 3 份 | 報到組 | A4 列印，勾選欄留白 |
| 紙本 Agenda 時間表 | 10 份 | 總召 | 可供各組參考 |
| 白板 / 白板筆 | 1 組 | 場佈組 | Q&A 與公告用 |
| 便利貼 | 1 包 | 場佈組 | 匿名提問用 |
| 4G/5G 熱點裝置 | 1 台 | 技術支援 | 備用網路 |
| 充電線 / 行動電源 | 若干 | 技術支援 | demo 裝置供電 |

---

## 6. 備案啟動判斷流程

```
問題發生
  │
  ├─ AI 不可用？
  │   └─ 啟動 Part 1 備案（加速人工 Q&A 回覆）
  │
  ├─ Supabase 不可用？
  │   └─ 啟動 Part 2 備案（紙本流程）
  │
  ├─ 網路完全中斷？
  │   └─ 啟動 Part 3 備案（完整離線）
  │
  └─ Vercel 不可用？
      └─ 嘗試回退部署；若失敗啟動 Part 3 備案
```

---

## 7. 活動結束後

- 若活動期間使用了紙本備案，需在網路 / 系統恢復後補登資料
- 紙本名單由報到組保管至少一週
- 檢討備案啟動原因，記錄在事後回顧中
