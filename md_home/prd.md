# **PRD：戶外工作坊即時活動 Web App**

## **1\. 產品名稱**

**Field Flow（暫名）**  
副標：一日戶外工作坊即時活動助手

---

## **2\. 背景與問題定義**

本產品是為一場 **單日制戶外 coding 工作坊** 所設計的活動專用 Web App。活動約從上午 9:30 報到、10:00 開始、20:00 結束，參加者約 100 人，工作人員約 12 人，活動節奏約每 30–60 分鐘切換一次階段，且現場多人流動、主要以手機使用為主。

目前活動現場存在以下問題：

### **2.1 學員端問題**

1. 不知道現在活動進行到哪一個階段  
2. 不知道此刻應該做什麼  
3. 不知道有問題時該問誰  
4. 常見問題反覆出現，但答案分散、缺乏統一入口  
5. 想提問時沒有清楚的追蹤方式

### **2.2 工作人員端問題**

1. 同樣的問題被重複詢問，增加回覆負擔  
2. 需要快速同步現場「目前進行中的階段」  
3. 需要掌握每位學員是否已報到、是否已領午餐  
4. 需要同時處理報到、午餐、Q\&A、活動資訊更新，若工具分散會增加操作成本

### **2.3 問題本質**

這不是一個通用活動平台，而是為 **單一活動、單次使用** 設計的現場營運工具。重點不是長期擴張，而是讓這場活動能夠順利運作，降低現場混亂與重工。

---

## **3\. 產品目標**

本產品的目標如下：

1. 讓學員在打開網站後，能快速知道「現在要做什麼」  
2. 提供統一的問題查詢與提問入口，降低現場重複問答  
3. 透過 AI \+ 既有 Q\&A，優先消化活動相關常見問題  
4. 讓工作人員在同一個後台完成活動營運相關操作  
5. 讓工作人員即時掌握報到與午餐領取狀態

## **5\. 目標用戶／使用者角色**

## **5.1 法法（學員）**

特徵：

* 約 100 人  
* 以手機瀏覽器使用為主  
* 需先完成輕登入  
* 主要需求是獲得即時活動資訊、查問題、提問題、看自己狀態

可執行行為：

* 輕登入  
* 看 agenda  
* 看目前進行中的階段  
* 看公開 Q\&A  
* 搜尋問題  
* 問 AI  
* 正式發問  
* 查看自己的提問  
* 查看自己的午餐領取狀態

## **5.2 努努（工作人員）**

特徵：

* 約 12 人  
* 手機為主，也可能使用筆電  
* 使用共用密碼登入  
* 登入後需選擇自己的姓名，且可依現場需要隨時切換身份  
* 所有人權限相同

可執行行為：

* 查看個人化工作任務 agenda  
* 手動切換目前活動階段  
* 回覆 / 編輯 / 補充 Q\&A  
* 報到 / 撤回報到  
* 午餐領取標記  
* 查看 dashboard  
* 修改 agenda 與活動知識內容

## **6\. 使用情境與核心流程**

## **6.1 法法核心流程**

1. 學員開啟活動網站  
2. 輸入 **3 碼學員編號** 與 **信箱**  
3. 系統比對名單：  
   * 若編號不存在，提示「編號查詢不到」  
   * 若編號存在但信箱不符，提示「信箱不正確，請確認行前信收件信箱或洽現場努努」，並提供 Gmail 開啟按鈕  
4. 登入成功後，登入狀態維持到活動結束，學員也可主動登出  
5. 首頁顯示：  
   * 現在正在進行的 agenda 階段  
   * 目前任務／說明／注意事項  
   * 完整活動流程列表  
6. 學員可展開其他 agenda 階段查看內容  
7. 學員進入 Q\&A 區：  
   * 看所有問題  
   * 搜尋問題編號或關鍵字  
   * 篩選只看自己提問  
8. 學員可先向 AI 發問  
9. 若 AI 能回答，則直接回覆，並附 related questions  
10. 若 AI 無法 100% 確定，則明確表示不確定，並引導學員轉正式提問，且可預先整理提問文字  
11. 學員正式發問後，問題立即公開，並獲得問題編號（例如 Q001）  
12. 學員可在公開 Q\&A 中持續查看自己的問題與工作人員回覆  
13. 學員可查看自己的午餐領取狀態

## **6.2 努努核心流程**

1. 工作人員進入努努後台  
2. 輸入共用密碼 `0012`  
3. 登入成功後可持續保持登入狀態，直到使用者主動登出  
4. 選擇自己的姓名，且登入後可隨時切換努努身份  
5. 進入同一個後台操作頁面  
6. 查看：  
   * 個人工作 agenda  
   * 目前活動階段  
   * dashboard  
   * Q\&A 狀態  
7. 執行以下操作：  
   * 手動切換目前進行中的 agenda  
   * 編輯 agenda 內容  
   * 編輯活動知識內容（Markdown）  
   * 回覆 / 補充 / 編輯 Q\&A  
   * 報到 / 撤回報到  
   * 標記午餐已領取  
8. 所有努努皆可看到 dashboard 與 Q\&A 狀態  
9. Q\&A 回覆需顯示回覆者姓名

## **7\. 產品價值主張**

### **對法法的價值**

* 一打開就知道現在要做什麼  
* 有問題時不用到處找人  
* 可以先透過 AI 與公開 Q\&A 自助解決  
* 自己提的問題可以追蹤  
* 能確認自己的午餐狀態

### **對努努的價值**

* 減少重複回答相同問題  
* 能即時同步活動節奏  
* 能快速掌握學員報到與午餐狀況  
* 能在單一後台完成現場大部分操作  
* 降低多工具切換與資訊散落風險

## **8\. 功能需求清單**

## **8.1 法法端功能**

### **A. 輕登入**

* 欄位：  
  * 學員編號（固定 3 碼）  
  * 信箱  
* 驗證邏輯：  
  * 先比對學員編號  
  * 再比對該編號對應信箱  
* 錯誤提示：  
  * 編號不存在：顯示「編號查詢不到」  
  * 信箱不符：顯示「信箱不正確，請確認行前信收件信箱或洽現場努努」  
* 提供 Gmail 開啟按鈕  
* 登入狀態維持到活動結束  
* 提供登出功能

### **B. Agenda 顯示**

* 首頁需明顯顯示：  
  * 現在正在進行中的階段  
* 當前階段顯示完整資訊：  
  * 時間  
  * 階段名稱  
  * 任務  
  * 任務說明  
  * 注意事項  
* 其他階段預設僅顯示：  
  * 時間  
  * 階段名稱  
* 使用者可點擊展開查看完整內容

### **C. Q\&A**

* 查看所有公開問題  
* 查看未回答問題  
* 搜尋：  
  * 關鍵字  
  * 問題編號  
* 可切換：  
  * 所有問題  
  * 我的問題  
* 問題詳情支援滿版 popup  
* 每則問題需顯示：  
  * 問題編號  
  * 發問者編號  
  * 問題內容  
  * 狀態  
  * 回覆內容  
  * 回覆者姓名  
  * 回覆時間

### **D. 正式提問**

* 發問時需綁定目前登入學員編號  
* 問題送出後立即公開  
* 系統自動產生問題編號（例如 Q001）  
* 問題可有多則回覆／補充  
* 問題狀態至少包含：  
  * 待回答  
  * 已回答

### **E. AI 助手**

* UI 位於右下角  
* 只能回答活動相關問題  
* 回答前應優先查詢：  
  * Agenda  
  * 活動知識庫  
  * 既有 Q\&A / 回答  
* 若發現已有相似問題，優先提供既有答案  
* 應顯示 related questions  
* 若無法有把握回答，需明確表達不確定  
* 不確定時引導使用者轉正式提問  
* 可預先整理提問內容供使用者送出

### **F. 個人狀態**

* 法法可看到自己的午餐領取狀態

---

## **8.2 努努端功能**

### **A. 後台登入**

* 使用共用密碼登入  
* 密碼為 `0012`  
* 登入成功後可持續保持登入狀態，直到使用者主動登出  
* 可在同一裝置隨時切換不同努努身份，無需重新輸入密碼

### **B. 努努身份選擇**

* 登入後需選擇姓名，且可隨時切換姓名／身份  
* 選定姓名後，系統顯示該努努對應的任務 agenda

### **C. 工作人員 Agenda**

* 可查看所有 agenda  
* 額外顯示個人相關資訊：  
  * 負責任務  
  * 位置  
  * 需注意的突發狀況  
* 可手動指定目前進行中的 agenda 階段  
* 可修改 agenda 內容  
* 編輯方式為 Markdown 編輯器

### **D. Q\&A 管理**

* 查看所有問題  
* 篩選待回答問題  
* 回覆問題  
* 補充問題回答  
* 編輯既有回答  
* 回覆需顯示回覆者姓名

### **E. 報到管理**

* 透過輸入學員編號完成報到  
* 可撤回報到狀態  
* 可查詢學員資料（至少含姓名、信箱、飲食別）

### **F. 午餐管理**

* 透過輸入學員編號標記已領午餐  
* 可查看學員飲食別：  
  * 葷  
  * 素

### **G. Dashboard**

需顯示至少以下即時數據：

* 已報到人數  
* 未報到人數  
* 已領午餐人數  
* 未領午餐人數  
* 問題總數  
* 待回答問題數

### **H. 活動知識內容管理**

* 努努可修改活動知識內容  
* 編輯方式為 Markdown 編輯器  
* AI 問答需使用最新內容作為知識來源之一

## **10\. 使用者故事（User Stories）**

### **法法**

1. 作為學員，我希望一登入就能看到目前活動進行到哪裡，這樣我知道現在該做什麼。  
2. 作為學員，我希望看到完整 agenda，這樣我能掌握整天流程。  
3. 作為學員，我希望能搜尋 Q\&A，這樣我不用重複發問。  
4. 作為學員，我希望先問 AI，這樣我能更快得到答案。  
5. 作為學員，我希望 AI 找不到答案時能幫我整理問題，這樣我能快速提交正式提問。  
6. 作為學員，我希望看到自己提過的問題，這樣我能追蹤回覆進度。  
7. 作為學員，我希望知道自己是否已領午餐，這樣我能確認自己的狀態。

### **努努**

1. 作為工作人員，我希望能登入後選擇自己的姓名，這樣我能看到自己該做的事。  
2. 作為工作人員，我希望能手動切換目前活動階段，這樣能即時同步現場狀態。  
3. 作為工作人員，我希望能在同一後台回覆 Q\&A，這樣能減少重複回答。  
4. 作為工作人員，我希望能快速用學員編號完成報到與午餐標記，這樣能提升現場效率。  
5. 作為工作人員，我希望在 dashboard 一眼看到未報到與未領午餐名單趨勢，這樣能及時處理現場問題。  
6. 作為工作人員，我希望能直接修改 agenda 與活動知識，這樣能應對臨場變更。

## **11\. 驗收標準（Acceptance Criteria）**

## **11.1 法法端**

### **輕登入**

* 輸入格式正確時，系統可驗證 3 碼學員編號與信箱  
* 不存在的編號會顯示「編號查詢不到」  
* 編號存在但信箱不符時會顯示錯誤提示與 Gmail 開啟按鈕  
* 登入成功後，使用者於同裝置可持續登入至活動結束，且可主動登出

### **Agenda**

* 首頁可明確顯示目前進行中的 agenda 階段  
* 當前階段顯示完整內容  
* 其他階段預設收合，點擊可展開  
* 法法在手機上可順利閱讀與操作

### **Q\&A**

* 法法可看到所有公開問題  
* 法法可搜尋關鍵字與問題編號  
* 法法可切換只看自己提問  
* 問題詳情可用滿版 popup 顯示  
* 發問後問題立即公開並產生唯一問題編號

### **AI 助手**

* AI 可根據 agenda / 活動知識 / 既有 Q\&A 回答活動相關問題  
* 找到相似問題時，優先提供既有答案與 related questions  
* AI 不確定時，會明確說明並引導正式提問  
* AI API 不可用時，畫面顯示「AI 目前正在午休中」

### **個人午餐狀態**

* 法法可看到自己的午餐領取狀態

## **11.2 努努端**

### **登入與身份選擇**

* 輸入正確共用密碼可登入後台  
* 登入後可選擇姓名  
* 選擇姓名後可顯示個人任務 agenda  
* 同裝置可隨時切換努努身份，且切換時無需重新輸入密碼

### **Agenda 管理**

* 努努可手動切換目前進行中的 agenda 階段  
* 切換後法法首頁應同步顯示新階段  
* 努努可透過 Markdown 編輯器修改 agenda

### **Q\&A 管理**

* 努努可查看全部問題與待回答問題  
* 努努可新增回覆、補充回覆、編輯回覆  
* 系統需紀錄並顯示回覆者姓名

### **報到與午餐**

* 努努可透過輸入學員編號完成報到  
* 努努可撤回報到  
* 努努可標記午餐已領取  
* 努努可看到學員飲食別

### **Dashboard**

* Dashboard 至少顯示六項指標：  
  * 已報到  
  * 未報到  
  * 已領午餐  
  * 未領午餐  
  * 問題總數  
  * 待回答數

## **11.3 系統與韌性**

* 上線前可成功匯入 CSV 名單與 agenda  
* 系統可預載重要內容，降低弱網環境下重複載入需求  
* 即時資料同步延遲時，顯示重新整理提示  
* 後端不可用時，團隊可切換人工備案流程

### **必達指標**

5. 工作人員可在手機上完成大部分操作

### **加分指標**

1. 法法能順利透過搜尋與我的問題功能追蹤提問  
2. 努努可快速適應同一後台，無需額外培訓  
3. 現場 agenda 調整後，法法端能快速同步理解最新狀態

## **13\. 風險、限制與依賴**

## **13.1 風險**

1. **臨時上線風險高**  
   功能範圍集中且明確，但時間非常緊迫，實作需優先保障主流程可用。  
2. **共用密碼安全性有限**  
   努努登入採共用密碼，安全性較低，但屬本次可接受取捨。  
3. **學員登入依賴事前名單正確性**  
   若學員編號與信箱資料錯誤，將導致登入失敗並需要現場人工處理。  
4. **AI 回答品質依賴知識整理品質**  
   若 agenda、活動知識與 Q\&A 不完整，AI 可用性會下降。  
5. **弱網狀態下互動功能可能受影響**  
   雖可透過預載降低問題，但發問、同步狀態更新仍依賴網路。

## **13.2 限制**

* 僅支援單一活動  
* 不做正式帳號系統  
* 不做完整離線模式  
* 不做權限分級  
* 不做問題審核與排序

## **13.3 依賴**

* Supabase 後端與資料庫  
* GitHub / Vercel 部署  
* CSV 名單與 agenda 準確性  
* 活動知識庫內容整理完成  
* ChatGPT API 可用性

## **14\. 開放問題（Open Questions）**

以下問題雖不影響 PRD 成立，但進入 Spec 前仍建議明確化：

1. 學員若忘記編號但知道姓名，是否要提供努努端姓名搜尋功能  
2. 問題狀態是否只需要「待回答 / 已回答」，或仍要保留更多狀態欄位供後續擴充  
3. Q\&A 的排序規則是依最新發問、最新回覆，或待回答優先  
4. AI related questions 的呈現方式要採列表、卡片，或直接可點擊跳轉  
5. 預載內容的範圍與快取策略要做到哪個程度  
6. Markdown 編輯器是否需支援版本還原或僅即時覆蓋

* Q\&A 狀態流轉
